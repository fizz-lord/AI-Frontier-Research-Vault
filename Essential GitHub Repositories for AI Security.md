# Essential GitHub Repositories for AI Security

- [Azure/PyRIT](https://github.com/Azure/PyRIT) - The Python Risk Identification Tool for generative AI. Open source AI red teaming framework used by Microsoft to build attack playbooks, automate probes against LLM apps, and generate reports.
- [NVIDIA/garak](https://github.com/NVIDIA/garak) - LLM vulnerability scanner. Automatically probes models for jailbreaks, prompt injection, data leakage, and toxic outputs.
- [promptfoo/promptfoo](https://github.com/promptfoo/promptfoo) - Test prompts, agents, and RAG pipelines with built-in security checks. Plugs into CI/CD so security testing runs on every release.
- [protectai/llm-guard](https://github.com/protectai/llm-guard) - Security toolkit for LLM interactions. Provides prompt sanitisation, PII detection, and harmful-content filtering at runtime.
- [NVIDIA-NeMo/Guardrails](https://github.com/NVIDIA-NeMo/Guardrails) - Open source toolkit for adding policy-driven guardrails to conversational AI. Define rules and flows for what an assistant can and cannot do.
- [meta-llama/PurpleLlama](https://github.com/meta-llama/PurpleLlama) - Meta's safety suite. Includes Llama Guard for input/output classification and tools for jailbreak detection.
- [Trusted-AI/adversarial-robustness-toolbox](https://github.com/Trusted-AI/adversarial-robustness-toolbox) - Adversarial Robustness Toolbox for classic adversarial machine learning attacks and defences, including evasion, poisoning, and extraction.
- [Giskard-AI/giskard-oss](https://github.com/Giskard-AI/giskard-oss) - Open source evaluation and testing library for LLM agents. Scans models for bias, leakage, and robustness issues.
- [orca-ai/awesome-llm-security](https://github.com/orca-ai/awesome-llm-security) - Community curated index of LLM security tools, papers, and attack research.
- [gresham/llm-security](https://github.com/gresham/llm-security) - Research repository on new ways of breaking application-integrated LLMs, including indirect prompt injection.
